Sprint Retrospective

What went well during the sprint cycle?:

  1. We finished the required project!
  2. We finished all the stories and passed all the mocha test
  3. We worked on our own branches for each story and created pull requests which made it much easier to manage the project without merge conflicts
  4. We were able to implement Google Analytics A/B tests using documentation on its website, though it took up a lot of time and was not very straightforward to implement and test
What went wrong during the sprint cycle?
  1. The Google analytics were a challenge because we didn’t know how to use them and the documentation was not very straightforward
  2. The RSVP button added in each event box in the homepage could not be a button with a link. Whenever we added a link, it would fail the mocha tests. So we had to change the RSVP to plain text (refer to Experiment 1 images below)
  3. The Thanksgiving holiday prevented us from meeting as usual during the week, but we managed to work virtually from different locations
  4. We were very close to finishing all the tasks down to 6 story points, but we decided to add more tasks (as seen in the spike on the burndown chart) such as the style for the donation and the event detail pages etc.
What could we do differently to improve?
