  3. Narrative of what has been accomplished since last report including any notable challenges or roadblocks encountered:

    • Since the last sprint, we have been able to finish all the user stories and have passed all 43 mocha tests.
    • The process involved, validation checks on event creation page, additions in the code to ensure that if entered details for a new event meet the required criteria, a new event-page is created with a new id for the event. This steps required significant effort from the team as we didn’t have the requisite knowledge on syntax - (id[0-9]+)
    • Once the event page was generated, we created the rsvp button with requisite attributes, id and type
    • The rsvp can be used by anyone with a @yale id and gives an error for any non @yale id. Again, this required change of entered id to lowercase and we went to office hours to be able to understand the right syntax.
    • A major roadblock occurred when despite having been able to meet the requirements our mocha tests weren’t passing for rsvp function and significant time was spent in debugging the issue. Ultimately we realized that the only reason why we were failing the tests was a lower-case ‘s’ in our submit button.

  5. Narrative of what expected work before next report
